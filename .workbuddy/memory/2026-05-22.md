## 健康打卡 App Bug 修复

**问题**: 用户输入"16:22喝了500ml水"，AI 错误地将其拆分为"8:00 250ml + 10:00 250ml"，确认录入后记录时间错误。

**根因**:
1. `confirmRecord()` 用文本解析（正则）提取数据，脆弱且不支持时间戳
2. 水数据只存总量（数字），不存时间
3. AI 收到水记录后随意拆分多条并编造时间

**修复内容**:
1. `buildConfirmCard()`: 增加 `data-record` 属性存储完整结构化数据，特殊格式化水记录显示
2. `confirmRecord()`: 直接从 `data-record` JSON 属性读取结构化数据，不再依赖文本解析
3. `saveRecordToData()`: 水改为 `waterEntries` 数组格式 `[{time, amount}]`，支持追加多条
4. `refreshDashboard()`: 水量展示从 `waterEntries` 数组累加（兼容旧 `water` 数字格式）
5. System Prompt: 增加【饮水记录格式】规则，要求 AI 不要拆分、不要捏造时间
